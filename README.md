# Студент & Т — экосистема где студенты могут эффективно учиться

**Студент & Т** — это полнофункциональная веб-платформа, созданная для управления образовательным пространством. Проект объединяет в себе инструменты для организации мероприятий, бронирования помещений, общения в чате и на форуме, а также систему наставничества.

Проект разработан командой **GeekForce**.

---

##  Основные возможности

- **Пользователи и роли** : Поддержка трёх ролей — пользователь (`user`), наставник (`mentor`) и администратор (`admin`) с разными уровнями доступа.
- **Мероприятия**: Создание, просмотр и управление событиями (хакатоны, лекции, воркшопы) с указанием цены, описания и иконки.
- **Бронирование помещений**: Гибкая система бронирования комнат с визуальной схемой мест (JSON-схема с поддержкой VIP-зон и мест для людей с ограниченными возможностями).
- **Система наставничества**: Возможность для пользователей становиться наставниками, управлять заявками и связываться со студентами.
- **Чат и форум**: Встроенный веб-сокет-чат для общения в реальном времени и форум для обсуждений.
- **Административная панель**: Отдельный интерфейс для администраторов с доступом к аналитике и управлению всеми сущностями системы.
- **Адаптивный интерфейс**: Фронтенд на базе **Next.js** обеспечивает удобную работу с любых устройств.

## 🛠 Технологический стек

| Компонент       | Технологии                                                                                             |
| :-------------- | :----------------------------------------------------------------------------------------------------- |
| **Бэкенд**      | [FastAPI](https://fastapi.tiangolo.com/) (Python 3.9+)                                               |
| **Фронтенд**    | [Next.js](https://nextjs.org/) (TypeScript, React)                                            |
| **База данных** | PostgreSQL (с использованием SQLAlchemy ORM)                                                     |
| **Аутентификация** | JWT-токены (библиотека `python-jose`) с хешированием паролей (`bcrypt`)                  |
| **WebSockets**  | Поддержка реального времени для чата                                                             |
| **Дополнительно** | `python-dotenv`, `psycopg2-binary`, `pydantic`, `websockets`                                    |

## 🏗 Установка и запуск

### 1. Клонирование репозитория

```
git clone https://github.com/Reneletco/StudentT.git
cd StudentT
```
2. Настройка бэкенда
Создайте и активируйте виртуальное окружение Python:
```
python -m venv venv
source venv/bin/activate      # для Linux/macOS
venv\Scripts\activate         # для Windows
```
Установите зависимости:
```
pip install -r requirements.txt
```
Создайте файл .env в корне проекта и заполните его своими данными:
```
DATABASE_URL = "postgresql://user:password@localhost:5432/studentt_db"
SECRET_KEY = "your_super_secret_key_here_change_it"
ALGORITHM = "HS256"
ACCESS_TOKEN_EXPIRE_MINUTES = 1440
```
Инициализируйте базу данных:
```
python init_db.py
```
Эта команда создаст все необходимые таблицы и добавит тестовые данные, включая учётную запись администратора: admin / admin.

Запустите сервер бэкенда:
```
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```
После запуска API будет доступно по адресу http://localhost:8000. Интерактивная документация (Swagger) будет доступна по адресу http://localhost:8000/docs.

3. Настройка фронтенда
Перейдите в директорию фронтенда:
```
cd frontend
```
Установите зависимости Node.js:
```
npm install
# или
yarn install
```
Соберите фронтенд для продакшена:
```
npm run build
```
После сборки статические файлы будут находиться в папке frontend/out. Бэкенд автоматически начнёт их раздавать, так как в main.py настроен StaticFiles.

Для разработки: вы также можете запустить фронтенд в режиме разработки с помощью npm run dev. В этом случае вам может потребоваться настроить проксирование API-запросов для избежания CORS-ошибок.

📚 API Эндпоинты (основные)
Аутентификация
```
POST /api/auth/register — регистрация нового пользователя.

POST /api/auth/login — вход и получение JWT-токена.

GET /api/auth/me — получение данных текущего пользователя.
```
Мероприятия
```
GET /api/events — получение списка всех мероприятий.

POST /api/events — создание нового мероприятия (только для админов и наставников).

GET /api/events/{event_id} — получение деталей конкретного мероприятия.
```
Бронирование
```
POST /api/bookings — создание нового бронирования.

GET /api/bookings/my — просмотр своих бронирований.
```
Чат
```
WebSocket /ws/{token} — подключение к чату в реальном времени.
```
Полный список эндпоинтов с примерами запросов и ответов доступен в Swagger-документации (/docs).
